Platform Architecture

End-to-end observable, reliable, and scalable

AirSense runs one continuous pipeline: MQTT ingestion to Redis Pub/Sub and WebSocket broadcast, then into dashboard and mobile clients.

Backend Services

FastAPI + MongoDB (motor) + JWT + MQTT worker + Redis cache/pubsub for real-time processing.

Dashboard Layer

React 19 + TypeScript + Vite + Tailwind + Recharts for live monitoring, alert history, and reporting.

ML Engine

Scikit-learn Isolation Forest with NumPy/Pandas for anomaly detection and decision support outputs.

Mobile and Notifications

Flutter + Provider + Dio + FCM for push notifications, live values, and field response workflows.

Digital Twin Engine

FastAPI backend + React/Three.js 3D layer for gas leak, fire, and evacuation simulations.

Data flow: ESP32 -> MQTT Broker -> Worker -> MongoDB + Redis -> WebSocket Gateway -> Dashboard/Mobile.
